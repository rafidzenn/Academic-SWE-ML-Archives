## The Software Development Life Cycle (SDLC)

The SDLC is the engineering blueprint for building software. It prevents architectural chaos and ensures the final product aligns with user requirements. 

1. **Requirement Analysis:** "What are we actually building, and why?" (Stakeholder interviews, feasibility studies).
2. **Design:** "How are we going to build it?" (System architecture, database schemas, API contracts).
3. **Implementation (Coding):** "Building it." (Writing the actual backend/frontend logic).
4. **Testing:** "Did we break anything?" (Unit tests, integration testing, User Acceptance Testing).
5. **Deployment:** "Putting it in the hands of the users." (Pushing to staging environments, then production).
6. **Maintenance:** "Fixing what breaks and iterating."

### Methodologies
* **Waterfall:** Strict, sequential approach. Step 1 must finish before Step 2. High risk if market needs change, but excellent for systems where failure is catastrophic (e.g., aerospace).
* **Agile:** Flexible, modern approach. Features are built in short sprints, deployed, and iterated upon based on continuous user feedback.
